# 04_BUILD_AREA.md

# 模块名称

Build Area

中文：

布阵区

棋盘区

放置区

---

# 模块定位

整个游戏最核心区域。

玩家所有决策最终都发生在这里。

包括：

```text
征兵
拖拽
布阵
合成
升级
构筑
战斗
```

玩家每局90%以上操作发生在此区域。

---

# 玩家认知目标

玩家进入游戏后5秒内必须理解：

```text
白色格子可以放兵

深红格子不能放兵

深红格子未来可以扩展

兵可以移动

兵可以合成

兵会自动攻击
```

# CELL状态

共有7种状态

---

# STATE_EMPTY

名称：

空格

表现：

```text
仅显示格子
```

数据：

```ts
{
 unit:null
}
```

---

# STATE_OCCUPIED

名称：

占用

表现：

```text
显示单位
显示等级
```

例如：

```text
刀兵 Lv2
```

---

# STATE_DRAG_ENTER

名称：

拖拽进入

触发：

```text
单位进入格子范围
```

表现：

```text
边框金色

发光

放大105%
```

持续：

```text
拖拽结束
```

---

# STATE_DRAG_LEAVE

名称：

拖拽离开

表现：

```text
恢复默认
```

---

# STATE_CAN_MERGE

名称：

可合成

条件：

```text

等级相同
```

表现：

```text

脉冲动画
```

频率：

```yaml
0.6秒
```

---

# STATE_SELECTED

名称：

表现：

```text

```

---

# STATE_LOCKED

名称：

表现：

```text

```

不可拖拽

结果：

```text

```

---

# 相同单位

结果：

```text

```

---

# 不同单位

结果：

```text
交换位置
```

---

# 合成逻辑

条件：

```text
单位类型相同

等级相同
```

示例：

```text
刀1 + 刀1
```

结果：

```text
刀2
```

---

# 设计原则

玩家必须始终看到：

```text
哪里能放兵

哪里不能放兵

哪里能合成

哪里可以扩展
```

不允许出现：

```text
格子边界不清晰

拖拽目标不明确

合成反馈不明显
```
